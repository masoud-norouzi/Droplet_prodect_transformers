\section{Purpose}

This note documents the junction branch-decision diagnostic and the follow-up
geometry-violation analysis.  The goal is to separate two related but distinct
failure modes:

\begin{itemize}
  \item \textbf{Decision error:} the model commits to the wrong outgoing branch
  at the junction.
  \item \textbf{Physical geometry error:} the predicted droplet footprint leaves
  the admissible channel mask, measured by ellipse outside-channel fraction.
\end{itemize}

The analysis uses saved validation rollout predictions from:
\begin{verbatim}
outputs/models/rollout_model_comparison/rollout_predictions.npz
\end{verbatim}

No model inference or retraining is involved.

\section{Junction Event Definition}

True junction events are identified from the true future trajectory only.  A
trajectory is counted as a true junction event when it has at least one valid
assignment to an explicitly allowed incoming branch before committing to an
outgoing branch.  The outgoing branch commitment rule is:

\begin{quote}
same outgoing branch assignment for 3 consecutive evaluable rollout steps.
\end{quote}

The outgoing branches are \texttt{left} and \texttt{right}.  The incoming branch
used in this validation set is \texttt{inlet}.  Arbitrary non-outgoing branches
are not treated as incoming, and a trajectory that begins already assigned to
\texttt{left} or \texttt{right} is not counted as a true junction event.

The corrected event finder identified 4,990 true junction events:
\begin{itemize}
  \item \texttt{inlet -> left}: 1,555 events
  \item \texttt{inlet -> right}: 3,435 events
\end{itemize}

\section{Catastrophic Channel Escape by Branch Decision}

The previous generic geometry-violation summaries used a low outside-channel
threshold and were not the right diagnostic for branch-decision interpretation.
They are therefore not reported here.  Instead, this section uses a stricter
catastrophic escape criterion:

\begin{quote}
predicted ellipse outside-channel fraction $\geq 0.95$ at any evaluable rollout
step after predicted branch commitment.
\end{quote}

This asks whether a committed branch prediction later becomes a complete channel
escape.  The analysis was run by the standalone utility:
\begin{verbatim}
python -m utils.junction_catastrophic_escape_table
\end{verbatim}

Under this stricter complete-escape definition, the geometry-aware model has no
catastrophic escapes among strict wrong-branch junction events.  This does not
mean that the geometry-aware model never produces visually catastrophic branch
tracking failures.  It means that, within the strict incoming-to-outgoing
junction-decision event set, its wrong-branch predictions did not reach
outside-channel fraction $\geq 0.95$ after predicted commitment.

\section{Why Track 3050 Is Not in the Strict Junction Table}

The geometry-aware video for track \texttt{3050} is a real catastrophic failure:
the true droplet is on the left branch, the prediction moves to the right, and
the predicted footprint eventually reaches outside-channel fraction 1.0.  The
specific video case is validation window 956, slot 5, track \texttt{3050}.

That case is not counted in the strict junction-decision table because the true
trajectory is already assigned to the left branch at rollout step 1.  It does
not contain an explicit \texttt{inlet -> left} transition inside the rollout
window.  Therefore it fails the strict true-event criterion:

\begin{quote}
explicit incoming branch assignment before outgoing branch commitment.
\end{quote}

Track \texttt{3050} should therefore be interpreted as a
\textbf{post-commitment branch tracking failure}, not as a strict
incoming-to-outgoing junction decision event.  This distinction matters:

\begin{itemize}
  \item \textbf{Junction decision failures:} the droplet enters from the inlet
  and the model chooses the wrong outgoing branch within the rollout window.
  \item \textbf{Post-commitment branch tracking failures:} the droplet is
  already on or near a branch at the start of the rollout, and the model later
  diverges into another branch or leaves the channel.
\end{itemize}

\section{Interpretation}

The decision and geometry diagnostics together show that branch choice and
complete channel escape are not the same error mode.  The history-based model
has the best branch-decision accuracy.  The Markovian model has the largest
catastrophic escape rate among wrong \texttt{left -> right} decisions.  The
geometry-aware model shows zero catastrophic escape among strict wrong-branch
junction events, but it can still fail catastrophically in post-commitment branch
tracking windows such as track \texttt{3050}.

The next useful diagnostic is therefore separate from the strict junction-entry
analysis: explicitly measure post-commitment branch tracking failures for
trajectories that begin already assigned to \texttt{left} or \texttt{right}.
